# طلب بناء WordPress Plugin: "مقياس الأسرة القدوة"

## السياق العام
موقع وردبريس لمركز "قيمة وقدوة" للتدريب (training.goodwaty.org.sa). المطلوب بلجن خاص بيه يقدم استبيان تفاعلي اسمه "مقياس الأسرة القدوة"، بيقيس مدى كون الأسرة "قدوة" عبر 4 محاور، وبيعطي نتيجة من 100 مع تصنيف وتوصية، وبيفتح قناة تواصل مباشرة مع الفريق لو النتيجة محتاجة تدخل.

## التقنيات المطلوبة (Strict Stack)
- **PHP** خالص لمنطق البلجن (WordPress Plugin Boilerplate Architecture)
- **لا** React / Vue / Angular / أي SPA framework
- **لا** Three.js أو أي 3D library (الاستبيان نص وفورمز بس، مفيش حاجة بصرية ثلاثية الأبعاد مطلوبة)
- Frontend: **Alpine.js** (خفيف، CDN أو مرفق محلي) + **Vanilla CSS** أو Tailwind utility classes مرفقة محليًا (بدون build step، الموقع على shared hosting بيئة محدودة)
- لا تتطلب npm/webpack/build process — الكود لازم يشتغل مباشرة من غير compile step
- متوافق مع WordPress 6.x وPHP 8.1+

## البنية المطلوبة للبلجن

### 1. Custom Database Table
بلجن activation hook ينشئ جدول مخصص (مش Custom Post Type، عشان حجم البيانات وسهولة الفلترة لاحقًا):

```sql
CREATE TABLE wp_aqs_responses (
  id BIGINT UNSIGNED AUTO_INCREMENT PRIMARY KEY,
  full_name VARCHAR(150) NOT NULL,
  email VARCHAR(150) NOT NULL,
  phone VARCHAR(30) NULL,
  answers LONGTEXT NOT NULL,      -- JSON: {question_id: score}
  axis_scores LONGTEXT NOT NULL,  -- JSON: {axis_1: total, axis_2: total, ...}
  total_score DECIMAL(5,2) NOT NULL,
  classification VARCHAR(50) NOT NULL, -- 'role_model' | 'on_the_way' | 'needs_consultation'
  ip_address VARCHAR(45) NULL,
  created_at DATETIME NOT NULL,
  contacted TINYINT(1) DEFAULT 0
);

CREATE TABLE wp_aqs_contact_requests (
  id BIGINT UNSIGNED AUTO_INCREMENT PRIMARY KEY,
  response_id BIGINT UNSIGNED NOT NULL,
  message TEXT NULL,
  preferred_contact_method VARCHAR(50) NULL,
  status VARCHAR(20) DEFAULT 'pending', -- pending | contacted | closed
  created_at DATETIME NOT NULL,
  FOREIGN KEY (response_id) REFERENCES wp_aqs_responses(id)
);
```

### 2. محاور الاستبيان (Seed Data ثابتة في PHP array أو JSON config)

استخدم هذا المحتوى بالضبط (4 محاور × 5 أسئلة = 20 سؤال):

**المحور 1: التأصيل القيمي والنمذجة السلوكية (البناء الداخلي)**
1. نحرص على تطبيق السلوك الإيجابي أمام الأبناء قبل أن نطلبه منهم
2. نربط المواقف اليومية بالقيم الإسلامية بطريقة غير متكلفة
3. نعتمد على الحوار والإقناع في غرس السلوكيات بدلاً من الأوامر الجافة والتلقين
4. نتجنب التناقض بين ما نأمر به الأبناء وبين ما نمارسه نحن كآباء وأمهات
5. يلتزم جميع أفراد الأسرة بقواعد واضحة وموحدة تنظم شؤون حياتهم اليومية ومتفق عليها بين أفرادها

**المحور 2: التماسك الأسري والمرونة النفسية (بيئة العلاقات)**
1. نخصص وقتاً يومياً للحوار الأسري الحر بعيداً عن المشتتات التقنية
2. نُدير الخلافات الزوجية والأسرية بعيداً عن مسامع الأبناء وبطريقة تحفظ الاحترام
3. نوفر للأبناء بيئة أمنة تتيح لهم مصارحتنا بأخطائهم دون خوف من العقاب القاسي
4. نتعامل مع الأزمات الطارئة بروح الفريق الواحد والمرونة العالية
5. نوزع المسؤوليات المنزلية بعدل بين أفراد الأسرة لتعزيز روح الانتماء والمشاركة

**المحور 3: الأسرة في العالم الرقمي (المواطنة الرقمية)**
1. نمتلك اتفاقية واضحة لأوقات وأماكن استخدام الأجهزة الذكية داخل المنزل
2. نناقش المحتوى الرقمي الذي يشاهده الأبناء بوعي ونحلله نقدياً لتعزيز حصانتهم الذاتية
3. نحرص كوالدين على ضبط أوقات استخدامنا لهواتفنا لنكون قدوة لأبنائنا في الاستخدام المتوازن للتقنية
4. نُوجّه الأبناء لاستخدام التقنية وأدوات الذكاء الاصطناعي في تطوير مهاراتهم النافعة
5. نحترم خصوصية الأبناء الرقمية مع توفير رقابة واعية مبنية على الثقة المتبادلة

**المحور 4: الأثر المجتمعي وتوجهات الرؤية (المجتمع الحيوي)**
1. نمارس سلوكيات المحافظة على البيئة وترشيد الاستهلاك والموارد كجزء من مسؤوليتنا
2. نشجع أبناءنا على المشاركة في المبادرات التطوعية وخدمة المجتمع المحيط
3. نربط طموحات الأبناء الدراسية والمهنية باحتياجات التنمية بالمملكة العربية السعودية ورؤيتها المستقبلية
4. نحافظ على علاقات إيجابية وفعالة مع الجيران والأقارب
5. نغرس في الأبناء مبادئ التسامح واحترام التنوع والتعايش السلمي مع الآخرين

كل سؤال له 5 اختيارات بنفس الترتيب: **دائماً (5) / غالباً (4) / أحياناً (3) / نادراً (2) / أبداً (1)**

### 3. منطق الحساب
- أقصى نتيجة ممكنة: 20 سؤال × 5 = 100 نقطة (مجموع مباشر، بدون أوزان مختلفة)
- التصنيف:
  - **80-100**: "أسرة قدوة" (لون أخضر، رسالة تهنئة)
  - **69-79**: "في الطريق إلى أن تكون قدوة" (لون أصفر/برتقالي، رسالة تشجيعية + اقتراح استشارة اختيارية)
  - **أقل من 69**: "تحتاج إلى جلسات علاجية واستشارات" (لون أحمر فاتح — متجنبًا الصدمة، رسالة دافئة وغير قاسية + دعوة واضحة للتواصل)

⚠️ **مهم جدًا للأسلوب**: الرسائل لازم تكون **بنّاءة ومتعاطفة، مش مُحبِّطة أو حكمية**. الأسرة اللي بتاخد نتيجة منخفضة جايه تطلب مساعدة، مش تتحط في خانة "فاشلة". اكتب الرسائل بصيغة دعم لا تشخيص.

### 4. تجربة المستخدم (Frontend Flow)
1. **صفحة/شورت كود الاستبيان**: `[aqs_survey]` يظهر فورم متعدد الخطوات (4 خطوات = 4 محاور، أو خطوة واحدة طويلة — قرر الأفضل لتجربة الموبايل مع progress bar)
2. **بداية الاستبيان**: حقل اسم + إيميل (مطلوبين)، مفيش تسجيل دخول مطلوب
3. **الأسئلة**: radio buttons أو cards قابلة للضغط، مع progress indicator واضح (مثلاً "السؤال 5 من 20")
4. **شاشة النتيجة**: 
   - دائرة/مؤشر بصري للنتيجة الكلية (CSS فقط، بدون 3D)
   - تفصيل بالنتيجة لكل محور (4 progress bars)
   - رسالة التصنيف
   - زر "تواصل معنا" يفتح فورم قصير (رسالة + وسيلة تواصل مفضلة)
5. عند submit فورم التواصل: إرسال إيميل تلقائي للفريق (باستخدام `wp_mail()`) فيه بيانات المستخدم + النتيجة + رسالته، وكمان إيميل تأكيد للمستخدم نفسه

### 5. لوحة التحكم Admin (WP Admin Menu)
- صفحة "النتائج" تعرض جدول كل المحاولات (اسم، إيميل، نتيجة، تصنيف، تاريخ) مع فلترة حسب التصنيف
- صفحة "طلبات التواصل" تعرض اللي طلبوا تواصل، مع تغيير status (pending/contacted/closed)
- صفحة إحصائيات بسيطة (Chart.js مرفق محليًا): توزيع التصنيفات، متوسط النتيجة لكل محور بمرور الوقت
- إمكانية تصدير النتائج لـ CSV

### 6. الأمان والحماية (مهم لأن مفيش تسجيل دخول)
- WordPress nonces على كل الفورمز
- Rate limiting بسيط بناءً على IP (مثلاً: منع أكثر من 3 محاولات إرسال خلال 10 دقائق من نفس IP) لمنع spam
- التحقق من صيغة الإيميل (server-side validation، مش JS فقط)
- Sanitize/Escape كل المدخلات حسب معايير WordPress (`sanitize_text_field`, `sanitize_email`, `esc_html`, إلخ)
- حماية من SQL Injection باستخدام `$wpdb->prepare()`
